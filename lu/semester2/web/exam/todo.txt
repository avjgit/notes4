========= HTML code (15%)
OK,DONE • The page has to be developed according to HTML 5 or XHTML 1.0 Strict standard. DOCTYPE has to be declared and the document must conform to it.
later   • The document must validate using the W3C validator (http://validator. w3.org).
OK,DONE • ID and class attributes must be used appropriately throughout the document.
OK,DONE • HTML elements must be used according to their semantics.
OK,DONE • HTML document must contain all of these element types:
OK,DONE     * headings (different levels)
OK,DONE     * lists (ordered and unordered)
OK,DONE     * basic text formatting (paragraphs, line breaks, bold, italics)
OK,DONE     * images
OK,DONE     * data input form elements (later accessed from JavaScript code)
========= CSS code (15%)
later   • The CSS code must be developed according to CSS 2.1 specification and must validate using the CSS validator (http://jigsaw.w3.org/css-validator). Stable CSS3 properties may be used as well, but vendor- specific markup should not be used.
OK,DONE • Style information must be stored in a .CSS file linked to the HTML document.
OK,DONE • A print version has to be developed where some parts (for instance, navigation) of the document are hidden. “Print preview”function can be used for testing.
OK,DONE • At least 10 selectors have to be used in CSS document
OK,DONE • Some part of the document must demonstrate how two or more classes can be formatted using CSS
========= Data input form and data validation using JavaScript (10%)
OK,DONE • Validation of mandatory fields – a warning should be shown if some of the fields are not filled;
OK,DONE • Data type validation (for instance, if the value entered in date field is actually a date; or if a numeric field contains a number)
OK,DONE • Data range validation (for instance, if the number entered in ”month” field is between 1 and 12; or if value in salary field is positive.)
OK,DONE • In case of validation errors, a message explaining the problem has to be presented to the user. The form must not be submitted in this case. If the form is filled correctly, it should be submitted to http://naivist.net/form/.
